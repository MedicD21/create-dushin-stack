# Template File Map

This page documents the primary generated files for each template.

Notes:
- Some files vary by options (`--typescript`, `--tailwind`, `--src-dir`, `--starter-folders`).
- External generators (`create-next-app`, `create-vite`) may add additional framework-owned files over time.

## `next-app`

Generated by `create-next-app`, then customized.

Core scaffold additions/overrides:
- `.gitignore`
- `README.md`
- `AGENTS.md`
- `app/page.tsx` or `src/app/page.tsx` (or pages equivalent), replaced with starter content
- Starter files when `--starter-folders` is enabled:
- `src/lib/cn.ts` or `lib/cn.ts`
- `src/components/button.tsx` or `components/button.tsx`
- `src/hooks/README.md` or `hooks/README.md`
- `src/components/README.md` or `components/README.md`

## `vite-react`

Base from `create-vite --template react(-ts)` plus custom config.

Core scaffold additions/overrides:
- `.gitignore`
- `README.md`
- `vite.config.ts` or `vite.config.js`
- `src/index.css`
- `src/App.tsx` or `src/App.jsx`
- `src/main.tsx` or `src/main.jsx` updated to import `./index.css`
- `tsconfig.app.json` or `tsconfig.json` updated with alias paths (TS)
- `jsconfig.json` created for alias paths (JS)
- `package.json` updated with Tailwind dev deps when `--tailwind` is enabled
- Starter files when `--starter-folders` is enabled:
- `src/lib/cn.ts` or `src/lib/cn.js`
- `src/components/button.tsx` or `src/components/button.jsx`
- `src/hooks/README.md`
- `src/components/README.md`

## `vite-router-query`

Same Vite base as `vite-react`, with router/query wiring.

Core scaffold additions/overrides:
- All `vite-react` files, plus:
- `src/main.tsx` or `src/main.jsx` replaced with `BrowserRouter` and `QueryClientProvider`
- `src/App.tsx` or `src/App.jsx` replaced with route-based starter UI
- `package.json` dependencies include:
- `react-router-dom`
- `@tanstack/react-query`

## `node-api-hono`

Created directly by this CLI (no external app generator).

Generated files:
- `.gitignore`
- `README.md`
- `package.json`
- `tsconfig.json`
- `src/index.ts`

## `monorepo-web-ui`

Created directly by this CLI as a workspace starter.

Generated files:
- `.gitignore`
- `README.md`
- `package.json` (workspace root)
- `pnpm-workspace.yaml`
- `apps/web/package.json`
- `apps/web/tsconfig.json`
- `apps/web/vite.config.ts`
- `apps/web/index.html`
- `apps/web/src/main.tsx`
- `apps/web/src/App.tsx`
- `apps/web/src/index.css`
- `packages/ui/package.json`
- `packages/ui/tsconfig.json`
- `packages/ui/src/index.tsx`

## `ios-swiftui`

Created directly by this CLI as a native iOS SwiftUI starter.

Requirements: Xcode 15+, [XcodeGen](https://github.com/yonaskolb/XcodeGen)

Generated files:
- `README.md`
- `project.yml` (XcodeGen config)
- `<ProjectName>/Info.plist`
- `<ProjectName>/App/<ProjectName>App.swift`
- `<ProjectName>/Views/ContentView.swift`

After scaffolding:
```bash
brew install xcodegen
xcodegen generate
open <ProjectName>.xcodeproj
```

## `react-capacitor`

Same Vite base as `vite-react`, with Capacitor wired for iOS and Android.

Core scaffold additions/overrides:
- All `vite-react` files, plus:
- `capacitor.config.ts`
- `package.json` dependencies:
  - `@capacitor/core`
  - `@capacitor/ios`
  - `@capacitor/android`
- `package.json` devDependencies:
  - `@capacitor/cli`
- `package.json` scripts:
  - `cap:sync`
  - `cap:open:ios`
  - `cap:open:android`

## `plugin-file`

Created from a local JSON file via `--template-file`.

Template schema:
- `files`: map of relative paths to file content
- `packageJson`: optional object written to `package.json`

Generated files:
- Every file key under `files`
- Optional `package.json` from `packageJson`
